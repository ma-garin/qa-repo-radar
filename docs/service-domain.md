# Service Domain

This document defines the QA service domains used by QA Repo Radar.

## Target business context

The target context is a third-party verification and QA service company. The tool should support service delivery, proposal preparation, technical research, internal standardization, and capability development.

## Domains

### Test design support

Repositories that help create, review, generate, or improve test cases and test conditions.

Examples:

- pairwise testing
- decision table testing
- boundary value support
- model-based testing
- test case generation
- coverage modelling

### Test automation

Repositories that help automate test execution or test workflows.

Examples:

- Playwright utilities
- Selenium utilities
- Cypress plugins
- pytest plugins
- test orchestration

### API testing

Repositories useful for API verification.

Examples:

- contract testing
- OpenAPI-based testing
- schema validation
- REST API test generation

### E2E testing

Repositories useful for browser or user journey verification.

Examples:

- browser automation
- visual regression
- cross-browser execution
- test evidence capture

### Quality visualization

Repositories that help explain quality status, risk, coverage, defects, or trends.

Examples:

- test dashboards
- defect analytics
- coverage reports
- flaky test visualization

### Verification PMO

Repositories that support project-level quality governance.

Examples:

- issue analytics
- project metrics
- risk dashboards
- progress reporting

### Agile QA

Repositories useful for shift-left, continuous testing, BDD, or agile team quality practices.

Examples:

- BDD tooling
- acceptance criteria support
- CI quality gates
- continuous testing utilities

### Exploratory testing support

Repositories that help plan, execute, record, or summarize exploratory testing.

Examples:

- session-based testing
- test charters
- evidence capture
- exploratory test notes

### Compatibility and cross-browser testing

Repositories useful for device, browser, OS, or environment compatibility verification.

Examples:

- browser matrix testing
- device farm utilities
- responsive testing

### Non-functional testing

Repositories useful for performance, security, accessibility, reliability, or usability testing.

Examples:

- load testing
- accessibility testing
- OWASP-related testing
- reliability testing

### AI quality assurance

Repositories useful for testing, evaluating, or governing AI systems.

Examples:

- LLM evaluation
- prompt evaluation
- hallucination checks
- model robustness testing
- AI safety evaluation

### QA process improvement

Repositories useful for maturity assessment, process diagnosis, standardization, or improvement planning.

Examples:

- QA metrics
- test maturity models
- process health checks
- engineering productivity metrics
